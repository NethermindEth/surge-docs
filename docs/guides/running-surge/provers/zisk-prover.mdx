# Zisk Prover Setup

Raiko uses the Zisk GPU backend to generate ZK proofs for Surge blocks. This guide walks through setting it up.

## Prerequisites

- A machine with a CUDA-capable NVIDIA GPU (RTX 5090, L40, or similar)
- Docker and Docker Compose
- Git

## 1. Install CUDA and NVIDIA Container Toolkit

### Install NVIDIA Drivers

Ensure you have the latest NVIDIA drivers installed:

```bash
# Check if NVIDIA drivers are installed
nvidia-smi
```

If not installed, follow the [NVIDIA Driver Installation Guide](https://docs.nvidia.com/datacenter/tesla/driver-installation-guide/index.html) for your platform.

### Install NVIDIA Container Toolkit

You need the NVIDIA Container Toolkit so Docker containers can use the GPU:

<Tabs>
  <TabItem value="ubuntu" label="Ubuntu/Debian" default>

```bash
# Add the NVIDIA Container Toolkit repository
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
  sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list

# Install the toolkit
sudo apt-get update
sudo apt-get install -y nvidia-container-toolkit

# Configure Docker to use the NVIDIA runtime
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

  </TabItem>
  <TabItem value="centos" label="CentOS/RHEL">

```bash
# Add the NVIDIA Container Toolkit repository
curl -s -L https://nvidia.github.io/libnvidia-container/stable/rpm/nvidia-container-toolkit.repo | \
  sudo tee /etc/yum.repos.d/nvidia-container-toolkit.repo

# Install the toolkit
sudo yum install -y nvidia-container-toolkit

# Configure Docker to use the NVIDIA runtime
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

  </TabItem>
</Tabs>

Verify the installation:

```bash
docker run --rm --gpus all nvidia/cuda:12.0.0-base-ubuntu22.04 nvidia-smi
```

You should see your GPU listed in the output.

## 2. Clone and Configure Raiko

Clone the Raiko repository and checkout the real-time proving branch:

```bash
git clone https://github.com/NethermindEth/raiko.git
cd raiko
git checkout surge-alethia-real-time-proving
```

### Configure Chain Specs

Place the chain specification files in the appropriate config directory:

```bash
# For devnet
mkdir -p host/config/devnet

# Copy chain_spec_list.json from your protocol deployment
# (generated by surge-protocol-deployer.sh)
cp /path/to/chain_spec_list_default.json host/config/devnet/chain_spec_list.json
```

### Configure Raiko

Edit `host/config/devnet/config.json` with the following key parameters:

```json
{
  "address": "0.0.0.0:8080",
  "network": "your_l2_network_name",
  "concurrency_limit": 1,
  "l1_network": "your_l1_network_name"
}
```

- `address`: The address the prover server listens on
- `network`: The L2 network name (must match chain spec)
- `concurrency_limit`: Number of concurrent proving tasks (set to number of GPUs)
- `l1_network`: The L1 network name

:::note
The `chain_spec_list.json` file is generated during protocol deployment. Copy it as-is -- don't modify it unless you know what you're doing.
:::

## 3. Build and Run

### Docker (Recommended)

Build and run Raiko with GPU support:

```bash
docker compose -f docker-compose.gpu.yml up -d
```

:::tip Build Issues on RTX 5090
If you encounter build failures where `lib-float` fails (the `float.o` file disappears mid-archive during parallel compilation), set:
```bash
export CARGO_BUILD_JOBS=1
```
This forces sequential compilation and resolves the issue.
:::

### Verify

Check that the prover is running and accepting requests:

```bash
# Check container status
docker compose -f docker-compose.gpu.yml ps

# Check logs
docker compose -f docker-compose.gpu.yml logs -f

# Test the prover endpoint
curl http://localhost:8080/health
```

The prover exposes the real-time proving endpoint at `POST /v3/proof/batch/realtime`.

## 4. Integration with Catalyst

Once Raiko is running, point Catalyst at it:

```
RAIKO_URL=http://<raiko-host>:8080
```

When a new L2 block comes in, Catalyst will:
1. Construct a `RealTimeProofRequest` with block data and signal slots
2. Send it to Raiko's `/v3/proof/batch/realtime` endpoint
3. Receive the proof (~10-17 seconds)
4. Submit the atomic `propose()` transaction to the RealTimeInbox

## Known Issues

### Intermittent Zisk Failures

Zisk sometimes fails on certain transaction types, especially bridge-related ones. This is a known issue and the Zisk team is aware. Catalyst retries automatically on failure.

### GPU Memory

Heavy blocks (lots of transactions) need more VRAM. Keep an eye on GPU memory with:

```bash
nvidia-smi -l 1
```
